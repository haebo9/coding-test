## 1. 🔍 분석 (10분)
- **제약 & 복잡도**:
   - N <= 1,000,000 -> O(N)
- **유형 & 자료구조**: Stack
- **핵심 로직 (Key Idea)**: 
   1. [전처리] 
      - 빈 stack(list) 정의 
   2. [메인 로직] (예외/탈출 조건 포함)
      [수정 전] # 전체 중복 제거 로직
      - for 반복문으로 arr의 요소를 순회하면서 
         - stack 안에 없는 수면 stack에 넣는다. (append)
      - 반복문이 종료된 후의 리스트 확인

      [수정 후] # 전체 중복이 아닌 연속된 중복 만을 제거해야 함
      - for 반복문으로 arr의 요소를 순회하면서
         - stack이 비어있으면 stack에 push 
         - `stack이 비어있지 않으면 stack[-1]과 arr[i]를 비교하여`
            - 같으면 pass
            - 다르면 stack에 push
   3. [최종 결과] (return 값)
      - 반복문이 종료된 후의 리스트 반환
- **체크 포인트**:
   - [x] N=1 또는 0일 때 (최소 입력) -> 배열의 크기는 자연수(=1이상) 
   - [x] 데이터가 없는 경우 (Empty) 
   - [x] 모든 값이 동일한 경우 -> 상관없음
   - [x] 정답이 음수이거나 0일 가능성 -> 배열의 크기가 최소 1이므로 최소 1개의 요소가 정답 배열안에 들어감
   - [x] 값의 변화가 특정 범위를 벗어나면 안 되는가? (Max/Min Check)

## 2. 📚 학습 (5분)
> 이 문제를 통해 새로 알게 된 점이나 다시 확인한 개념을 정리합니다.

**① 오늘의 알고리즘/개념**
> 개념의 핵심 원리 한 줄 요약
- > **스택(Stack)을 활용한 연속 중복 제거**: 스택의 `Top`(마지막 요소) 접근은 **O(1)**이므로, 현재 값과 `stack[-1]`만 비교하면 효율적으로 연속된 중복을 제거할 수 있다. 반면 `if x in stack`을 사용하면 **O(N)** 탐색이 되어 전체 복잡도가 **O(N²)**이 되므로 주의해야 한다. 

**② Python Code Snippet (패턴 저장)**
> 나중에 써먹을 수 있는 핵심 코드 조각
```python
stack = []
for x in arr:
    # 스택이 비어있거나, 이전 값(Top)과 다르면 추가
    if not stack or stack[-1] != x:
        stack.append(x)
return stack
```
