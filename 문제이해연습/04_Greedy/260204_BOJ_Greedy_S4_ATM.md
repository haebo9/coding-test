> [문제 출처](https://www.acmicpc.net/problem/11399) <br/>

## 1. 🔍 분석 (10분)
- **제약 & 복잡도**:
   - N = 10^3 -> O(NlogN)
- **유형 & 자료구조**: Greedy
- **핵심 로직 (Key Idea)**: 
> 전략 : 빨리 끝나는 사람이 먼저 인출해야 뒷 사람의 합이 줄어듬
   1. [전처리] 
      - n 은 사람의 수 
      - p_lst 을 리스트의 형태로 입력 받음 
      - 작은 숫자가 앞으로 오도록 정렬
   2. [메인 로직] (예외/탈출 조건 포함)
      - i_sum = 0 # i 번쨰 까지의 합을 담을 객체 
      - result = [] # 부분 합을 저장할 리스트 
      - for문으로 p_lst의 앞에부터 순회하며 
         - i_sum 에 지금의 값(i)를 더하고 
         - result에 i_sum의 값을 추가 
   3. [최종 결과] (return 값)
      - result의 요소들의 합

- **체크 포인트**:
   - [x] N=1 또는 0일 때 (최소 입력)
   - [x] 데이터가 없는 경우 (Empty)
   - [x] 모든 값이 동일한 경우
   - [x] 정답이 음수이거나 0일 가능성
   - [x] 값의 변화가 특정 범위를 벗어나면 안 되는가? (Max/Min Check)
   - [x] 재귀 깊이 제한 (sys.setrecursionlimit)
   - [x] 리스트 복사 주의 (얕은 복사 vs 깊은 복사)

## 2. 📚 학습 (5분)
> 이 문제를 통해 새로 알게 된 점이나 다시 확인한 개념을 정리합니다.

**① 오늘의 알고리즘/개념**
> 그리디를 이용한 대기 시간 최소화 (SJF 스케줄링)
- 작업 시간이 짧은 순서대로 처리할수록, 뒤에 기다리는 사람들의 대기 시간이 누적해서 줄어든다.
- 따라서 **오름차순 정렬**이 핵심.
- 누적 합 문제로도 볼 수 있지만, 각 요소가 최종 결과에 몇 번 기여하는지를 파악하면 수학적으로 더 간단히 풀 수 있다.

**② Python Code Snippet (패턴 저장)**
> 반복 횟수 곱하기 (Mathematical Approach)
```python
# 1. 가장 작은 값부터 정렬
p_lst.sort()

# 2. 각 숫자가 더해지는 횟수를 곱해서 합산
# 첫 번째 숫자는 N번, 두 번째는 N-1번 ... 마지막은 1번 더해짐
n = len(p_lst)
result = sum(p * (n - i) for i, p in enumerate(p_lst))
```
